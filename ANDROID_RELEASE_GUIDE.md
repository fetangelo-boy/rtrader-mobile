## Build Options

### Option A: EAS Managed Build (Recommended)

**Advantages:**
- Automatic signing via EAS managed credentials
- Cloud build (no local Android SDK needed)
- Supports both APK and AAB
- Automatic version increment

**Requirements:**
- Expo account (https://expo.dev)
- EAS CLI installed (`npm install -g eas-cli`)
- No local Android SDK needed

**Steps:**

```bash
# 1. Login to Expo
eas login

# 2. Initialize EAS project (if not already done)
eas project:init

# 3. Build APK for testing
eas build --platform android --profile preview

# 4. Build AAB for Google Play Store
eas build --platform android --profile production

# 5. Download builds from EAS dashboard
# https://expo.dev/accounts/[your-account]/projects/rtrader-mobile
```

---

### Option B: Local Build (Requires Android SDK)

**Advantages:**
- Full control over build process
- No cloud dependency
- Faster iteration

**Requirements:**
- Android SDK (API 34+)
- Android NDK
- Java 17+
- Gradle

**Steps:**

```bash
# 1. Install Expo CLI
npm install -g expo-cli

# 2. Generate release APK
npx expo run:android --variant release

# 3. Generate release AAB
npx expo run:android --variant release --output-format aab

# 4. Builds will be in:
# - APK: android/app/build/outputs/apk/release/app-release.apk
# - AAB: android/app/build/outputs/bundle/release/app-release.aab
```

## Manual Signing (Optional)

If you want to use your own keystore instead of EAS managed credentials:

```bash
# 1. Create keystore (if you don't have one)
keytool -genkey -v -keystore rtrader-release.keystore \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias rtrader-key

# 2. Create eas.json with custom keystore
# See "Custom Signing" section below

# 3. Build with custom credentials
eas build --platform android --profile production
```

---

## Custom Signing Configuration

If using your own keystore, update `eas.json`:

```json
{
  "build": {
    "production": {
      "android": {
        "keystore": {
          "keystorePath": "./rtrader-release.keystore",
          "keystorePassword": "YOUR_KEYSTORE_PASSWORD",
          "keyAlias": "rtrader-key",
          "keyPassword": "YOUR_KEY_PASSWORD"
        }
      }
    }
  }
}
```

- **Use:** Direct installation, testing, beta distribution

- **Installation:** Cannot be installed directly (Play Store only)

## Google Play Store Submission

### Prerequisites

1. **Google Play Developer Account**
   - Cost: $25 one-time
   - Sign up: https://play.google.com/console

2. **App Signing Certificate**
   - Generated by EAS (recommended)
   - Or use your own keystore

3. **App Metadata**
   - App name: RTrader
   - Description: Трейдинговый супер-портал
   - Category: Finance
   - Content rating: Appropriate for all ages

### Submission Steps

1. **Create app in Play Console**
   - Go to https://play.google.com/console
   - Click "Create app"
   - Fill in app details

2. **Upload AAB**
   - Go to "Release" → "Production"
   - Click "Create new release"
   - Upload `app-release.aab`

3. **Add Store Listing**
   - App name, description, screenshots
   - Privacy policy URL
   - Contact email

4. **Set up pricing and distribution**
   - Free app
   - Available worldwide

5. **Submit for review**
   - Review takes 1-3 hours typically
   - Google will review for compliance

6. **Monitor review status**
   - Check Play Console for updates
   - Fix any issues if rejected

## Build Verification

### Pre-Build Checks

```bash
# 1. Check TypeScript compilation
pnpm check

# 2. Run tests
pnpm test

# 3. Verify environment variables
echo "EXPO_PUBLIC_SUPABASE_URL: $EXPO_PUBLIC_SUPABASE_URL"
echo "EXPO_PUBLIC_SUPABASE_ANON_KEY: $EXPO_PUBLIC_SUPABASE_ANON_KEY"
```

### Post-Build Checks

1. **APK Installation Test**
   ```bash
   adb install app-release.apk
   # Test on device: login → chats → messages → account
   ```

2. **AAB Validation**
   ```bash
   # Use bundletool to validate
   bundletool validate --bundle-path=app-release.aab
   ```

3. **Security Verification**
   - Check app is minified: `strings app-release.apk | grep -i "class.*activity"` (should show obfuscated names)
   - Verify signing: `jarsigner -verify -verbose app-release.apk`

## Troubleshooting

### Build Fails with "EAS Project ID not found"

**Solution:**
```bash
# Option 1: Use EAS
eas project:init

# Option 2: Use local build
npx expo run:android --variant release
```

### Build Fails with "Android SDK not found"

**Solution:**
```bash
# Install Android SDK
# macOS:
brew install android-sdk

# Linux:
sudo apt-get install android-sdk

# Windows:
# Download from https://developer.android.com/studio
```

### App crashes on launch

**Check:**
1. Supabase credentials are correct
2. Network connectivity
3. Logs: `adb logcat | grep RTrader`

### Push notifications not working

**Check:**
1. Device has Google Play Services
2. Notification permission granted
3. FCM token registered in backend
4. Supabase push_tokens table has entry
