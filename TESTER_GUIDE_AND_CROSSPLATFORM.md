# RTrader Mobile — Инструкция для тестировщика (iPhone) и кроссплатформенная стратегия

---

## Часть 1. Инструкция для тестировщика на iPhone

### Что нужно для тестирования

| Требование | Описание |
|---|---|
| Устройство | iPhone с iOS 16 или новее |
| Приложение | **Expo Go** (бесплатно в App Store) |
| Интернет | Wi-Fi или мобильный интернет |
| Telegram | Установлен на iPhone (для теста кнопки "Управление подпиской") |
| Время | 10-15 минут |

### Шаг 1. Установить Expo Go

1. Откройте **App Store** на iPhone
2. В поиске введите **Expo Go**
3. Нажмите **Загрузить** (приложение бесплатное, разработчик — Expo, Inc.)
4. Дождитесь установки

### Шаг 2. Открыть приложение RTrader

1. Откройте камеру iPhone (или приложение Expo Go)
2. Наведите камеру на QR-код, который вам пришлёт администратор
3. Появится уведомление — нажмите на него
4. Приложение RTrader загрузится в Expo Go (первая загрузка может занять 15-30 секунд)

**Альтернативный способ (без QR-кода):**
1. Откройте **Expo Go** на iPhone
2. Нажмите **"Enter URL manually"** (внизу экрана)
3. Введите ссылку, которую пришлёт администратор
4. Нажмите **Connect**

### Шаг 3. Вход в приложение

На экране логина введите данные, которые вам предоставит администратор:

1. **Email** — введите email-адрес
2. **Пароль** — введите пароль
3. Нажмите кнопку **"Войти"**

Если вам прислали **deep link** (ссылка вида `rtrader://login?email=...&password=...`), просто нажмите на неё — email и пароль заполнятся автоматически.

### Шаг 4. Что тестировать

#### 4.1. Экран "Чаты"
- [ ] Отображаются ли все чаты (должно быть 8 штук)
- [ ] Можно ли открыть любой чат нажатием
- [ ] Загружаются ли сообщения в чате
- [ ] Отображаются ли имена авторов сообщений
- [ ] Можно ли отправить сообщение (ввести текст → нажать кнопку отправки)
- [ ] Работает ли Reply (долгое нажатие на сообщение → ответить)
- [ ] После отправки сообщения — автоскролл к новому сообщению
- [ ] Клавиатура закрывается после отправки

#### 4.2. Экран "Аккаунт"
- [ ] Отображается имя пользователя и email
- [ ] Отображается статус подписки (Активна / дата окончания)
- [ ] Кнопка **"Изменить"** (профиль) — открывает экран редактирования
- [ ] Кнопка **"Управление подпиской"** — открывает чат с ботом в Telegram
- [ ] Кнопка **"Служба поддержки"** — открывает чат @rhodes4ever в Telegram
- [ ] Кнопка **"Выход"** — показывает диалог подтверждения, выходит из аккаунта
- [ ] Социальные ссылки (Telegram, VK, Сайт) — открываются корректно

#### 4.3. Экран "Профиль" (через кнопку "Изменить")
- [ ] Отображается аватар (или инициалы)
- [ ] Можно изменить имя пользователя
- [ ] Можно сменить пароль

#### 4.4. Общее
- [ ] Приложение не вылетает при навигации между экранами
- [ ] Тёмная/светлая тема корректно отображается
- [ ] Текст читаемый, шрифты нормального размера
- [ ] Кнопки реагируют на нажатие (визуальный feedback)
- [ ] Скорость загрузки приемлемая

### Шаг 5. Как сообщить о проблемах

Если что-то не работает, отправьте администратору:
1. **Скриншот** экрана с проблемой
2. **Описание**: что нажали, что ожидали, что произошло
3. **Модель iPhone** и **версию iOS** (Настройки → Основные → Об этом устройстве)

## Часть 2. Ответы на вопросы о кроссплатформенности

### Вопрос 1: Смогут ли подписчики пользоваться приложением на iPhone и MacBook?

**Короткий ответ: Да.**

Приложение RTrader написано на **React Native + Expo**, что означает, что один и тот же код работает на трёх платформах:

| Платформа | Способ доступа | Статус |
|---|---|---|
| **Android** | APK файл (установка напрямую) | ✅ Готово |
| **iPhone/iPad** | Expo Go или TestFlight или App Store | ✅ Код готов, нужна дистрибуция |
| **Web (MacBook, ПК)** | Браузер (Chrome, Safari, Firefox) | ✅ Работает уже сейчас |

Для **MacBook** подписчики могут использовать **веб-версию** через браузер — она уже работает. Все чаты, аккаунт, профиль — всё доступно. Отдельное macOS-приложение не требуется.

### Вопрос 2: Нужно ли переписывать код для iOS?

**Нет.** Код уже кроссплатформенный. React Native компилируется в нативный iOS-код автоматически. Никакой адаптации не требуется — приложение уже работает на iPhone через Expo Go.

### Вопрос 3: Подойдёт ли backend от Android-версии?

**Да, полностью.** Backend (сервер, база данных, API, Telegram-бот) — это единая серверная часть, которая не зависит от платформы клиента. Один backend обслуживает все платформы одновременно:

```
iPhone ──────┐
Android ─────┤──→ Единый Backend (API + Supabase + Telegram Bot)
Web (MacBook)┘
```

### Вопрос 4: Как распространять приложение без Apple Developer Account?

Это ключевой вопрос. Вот все варианты, отсортированные по простоте:

#### Вариант A: Веб-приложение (PWA) — без Apple Developer Account

| Параметр | Описание |
|---|---|
| Стоимость | **Бесплатно** |
| Сложность | Минимальная |
| Apple Developer Account | **Не нужен** |
| Установка для пользователя | Открыть ссылку в Safari → "Добавить на экран «Домой»" |
| Ограничения | Нет push-уведомлений на iOS, нет доступа к камере в некоторых случаях |

**Как это работает:** Вы даёте подписчикам ссылку (например, `https://rtradermob-gjsezgkc.manus.space`). Они открывают её в Safari и нажимают "Добавить на экран Домой". На рабочем столе появляется иконка приложения, которая выглядит и работает как обычное приложение. Это уже работает прямо сейчас.

**Для подписчика инструкция:**
1. Откройте ссылку в Safari
2. Нажмите кнопку "Поделиться" (квадрат со стрелкой вверх)
3. Выберите "На экран «Домой»"
4. Нажмите "Добавить"
5. Иконка RTrader появится на рабочем столе

#### Вариант B: Expo Go — без Apple Developer Account

| Параметр | Описание |
|---|---|
| Стоимость | **Бесплатно** |
| Сложность | Низкая (нужен QR-код) |
| Apple Developer Account | **Не нужен** |
| Установка для пользователя | Скачать Expo Go из App Store → отсканировать QR |
| Ограничения | Нужно установить Expo Go, приложение загружается каждый раз |

**Как это работает:** Подписчики устанавливают Expo Go (бесплатно) и сканируют QR-код. Приложение загружается внутри Expo Go. Подходит для тестирования и небольшой группы пользователей (до 50 человек).

#### Вариант C: Apple Developer Account ($99/год) — полноценная дистрибуция

| Параметр | Описание |
|---|---|
| Стоимость | **$99/год** |
| Сложность | Средняя |
| Apple Developer Account | **Нужен** |
| Установка для пользователя | TestFlight (до 10 000 тестеров) или App Store |
| Ограничения | Нет |

**Как это работает:** Вы регистрируете Apple Developer Account, собираете IPA через EAS Build (нажатие кнопки Publish), загружаете в TestFlight. Подписчики устанавливают TestFlight из App Store и получают приложение по ссылке-приглашению. Это самый профессиональный вариант.

### Рекомендация: оптимальная стратегия

Для вашего случая (VIP-клуб, ограниченное число подписчиков) оптимальная стратегия:

**Сейчас (бесплатно):**
- **Android** — APK файл (уже готов, раздаёте напрямую)
- **iPhone** — веб-версия (PWA) через ссылку в Safari
- **MacBook/ПК** — та же веб-версия через браузер

**Позже (когда будет 50+ подписчиков):**
- **Android** — Google Play ($25 разово) или APK
- **iPhone** — TestFlight через Apple Developer Account ($99/год)

### Вопрос 5: Как правильно делать изначально для всех платформ?

Вы уже всё сделали правильно. React Native + Expo — это и есть кроссплатформенный фреймворк. Один код → три платформы. Вот что у вас уже есть:

| Компонент | Кроссплатформенность |
|---|---|
| UI (экраны, кнопки, чаты) | ✅ Один код для iOS, Android, Web |
| Backend (API, база данных) | ✅ Единый сервер для всех платформ |
| Telegram-бот | ✅ Не зависит от платформы |
| Авторизация (Supabase Auth) | ✅ Работает везде |
| Deep links | ✅ Работают на iOS и Android |

**Единственное, что отличается** — это способ дистрибуции (как доставить приложение пользователю). Код менять не нужно.
