# Система регистрации и доступа подписчиков RTrader

**Автор:** Manus AI  
**Дата:** 24 апреля 2026  

---

## 1. Текущее состояние

Сейчас в приложении RTrader используется **Supabase Auth** с email/password аутентификацией. Пользователи хранятся в таблице `auth.users` Supabase, профили — в таблице `profiles`, подписки — в таблице `subscriptions`. Telegram-бот обрабатывает оплату и управляет подписками отдельно от приложения.

**Ключевая задача:** связать оплату через Telegram-бота с доступом в мобильное приложение, чтобы после оплаты подписчик автоматически получал учётные данные и мог войти в приложение.

---

## 2. Три варианта архитектуры

### Вариант A: Telegram-бот создаёт аккаунт и отправляет логин/пароль

Это самый простой и надёжный вариант. Telegram-бот после подтверждения оплаты вызывает Supabase Admin API, создаёт пользователя и отправляет ему логин/пароль прямо в Telegram.

**Схема потока:**

```
Подписчик → Оплата через Telegram-бота
                    ↓
        Бот подтверждает оплату
                    ↓
        Бот вызывает Supabase Admin API:
        1. Создаёт пользователя (email + пароль)
        2. Создаёт профиль в таблице profiles
        3. Создаёт подписку в таблице subscriptions
                    ↓
        Бот отправляет в Telegram:
        "Ваш логин: user@rtrader.com
         Ваш пароль: Xk9mP2qR
         Скачайте приложение: [ссылка]"
                    ↓
        Подписчик входит в приложение
```

| Параметр | Описание |
|----------|----------|
| **Email** | Бот запрашивает email у подписчика, либо генерирует автоматически (telegram_id@rtrader.com) |
| **Пароль** | Бот генерирует случайный пароль (8-12 символов) |
| **Подписка** | Бот создаёт запись в таблице `subscriptions` с датами начала/окончания |
| **Продление** | При продлении бот обновляет `expires_at` в таблице `subscriptions` |

**Плюсы:** простая реализация, полный контроль бота над аккаунтами, подписчику не нужно ничего делать кроме ввода логина/пароля.

**Минусы:** подписчик может забыть пароль (решается кнопкой "Забыли пароль?" или повторной отправкой через бота).

---

### Вариант B: Код доступа (invite code) вместо пароля

Telegram-бот генерирует одноразовый код-приглашение. Подписчик вводит этот код в приложении, и приложение автоматически создаёт аккаунт и привязывает подписку.

**Схема потока:**

```
Подписчик → Оплата через Telegram-бота
                    ↓
        Бот генерирует invite-код (например: RTR-A7K9-M2QP)
        Бот сохраняет код в таблицу invite_codes
                    ↓
        Бот отправляет в Telegram:
        "Ваш код доступа: RTR-A7K9-M2QP
         Скачайте приложение: [ссылка]
         Введите код при первом входе"
                    ↓
        Подписчик открывает приложение
        → Экран "Введите код доступа"
        → Приложение проверяет код через API
        → Создаётся аккаунт + подписка
        → Подписчик задаёт свой пароль
```

| Параметр | Описание |
|----------|----------|
| **Код** | Формат: `RTR-XXXX-XXXX`, срок действия 48 часов, одноразовый |
| **Регистрация** | Подписчик вводит код → задаёт email и пароль → аккаунт создан |
| **Подписка** | Привязывается автоматически при активации кода |
| **Продление** | Бот отправляет новый код или обновляет подписку напрямую |

**Плюсы:** подписчик сам выбирает пароль, не нужно передавать пароль через Telegram, более безопасно.

**Минусы:** дополнительный шаг для подписчика, нужна новая таблица `invite_codes`.

---

### Вариант C: Вход через Telegram (OAuth-подобный)

Подписчик нажимает кнопку "Войти через Telegram" в приложении. Приложение открывает Telegram Login Widget или deep link, Telegram подтверждает личность, и приложение проверяет подписку по `telegram_id`.

**Схема потока:**

```
Подписчик → Нажимает "Войти через Telegram" в приложении
                    ↓
        Открывается Telegram для подтверждения
                    ↓
        Приложение получает telegram_id + имя
                    ↓
        API проверяет: есть ли активная подписка
        для этого telegram_id?
                    ↓
        Да → Вход разрешён
        Нет → "Подписка не найдена. Обратитесь к боту @rtrader_bot"
```

| Параметр | Описание |
|----------|----------|
| **Идентификация** | По `telegram_id` — уникальный для каждого пользователя Telegram |
| **Связь** | Бот при оплате сохраняет `telegram_id` → `subscription` |
| **Пароль** | Не нужен — вход через Telegram |
| **Продление** | Бот обновляет подписку, приложение проверяет при каждом входе |

**Плюсы:** самый удобный для подписчика (никаких паролей), невозможно забыть логин, прямая связь Telegram ↔ приложение.

**Минусы:** сложнее в реализации (Telegram Login Widget работает только в браузере, для мобильного нужен Telegram Bot API + deep links), зависимость от Telegram API.

| Критерий | Вариант A (логин/пароль) | Вариант B (invite-код) | Вариант C (Telegram OAuth) |
|----------|--------------------------|------------------------|---------------------------|
| **Сложность для бота** | Средняя | Средняя | Высокая |
| **Сложность для приложения** | Низкая (уже есть) | Средняя | Высокая |
| **Удобство для подписчика** | Хорошее | Хорошее | Отличное |
| **Безопасность** | Средняя | Высокая | Высокая |

| **Передача доступа другим** | Возможна (email+пароль) | Затруднена (код одноразовый) | Невозможна |

## 4. Рекомендация: Вариант A + элементы B

Рекомендуется **комбинированный подход**: Telegram-бот создаёт аккаунт (Вариант A), но вместо отправки пароля в открытом виде отправляет **одноразовую ссылку для установки пароля** (элемент из Варианта B). Это сочетает простоту реализации с безопасностью.

### Рекомендуемый поток

```
1. Подписчик оплачивает через Telegram-бота
2. Бот запрашивает email (или использует telegram_id@rtrader.app)
3. Бот вызывает Supabase Admin API:
   - createUser(email, временный_пароль)
   - Создаёт профиль (username = имя из Telegram)
   - Создаёт подписку (plan, started_at, expires_at)
   - Сохраняет telegram_id в профиле
4. Бот отправляет подписчику:
   - Логин (email)
   - Временный пароль
   - Ссылку на скачивание приложения
5. Подписчик входит в приложение
6. Приложение проверяет подписку при каждом входе
```

### Что нужно реализовать

**На стороне Telegram-бота (Python/Node.js):**

```python
# Пример для Python (aiogram/python-telegram-bot)
import httpx
import secrets
import string

SUPABASE_URL = "https://xxx.supabase.co"
SUPABASE_SERVICE_KEY = "eyJ..."

async def create_subscriber(telegram_id: int, telegram_name: str, email: str, plan: str, days: int):
    """Создать подписчика после оплаты"""
    headers = {
        "apikey": SUPABASE_SERVICE_KEY,
        "Authorization": f"Bearer {SUPABASE_SERVICE_KEY}",
        "Content-Type": "application/json"
    }
    
    # 1. Генерируем пароль
    password = ''.join(secrets.choice(string.ascii_letters + string.digits) for _ in range(10))
    
    # 2. Создаём пользователя в Supabase Auth
    async with httpx.AsyncClient() as client:
        resp = await client.post(
            f"{SUPABASE_URL}/auth/v1/admin/users",
            headers=headers,
            json={
                "email": email,
                "password": password,
                "email_confirm": True,
                "user_metadata": {
                    "telegram_id": telegram_id,
                    "name": telegram_name
                }
            }
        )
        user = resp.json()
        user_id = user["id"]
    
        # 3. Создаём профиль
        await client.post(
            f"{SUPABASE_URL}/rest/v1/profiles",
            headers={**headers, "Prefer": "return=minimal"},
            json={
                "id": user_id,
                "username": telegram_name,
                "telegram_id": str(telegram_id),
                "email": email
            }
        )
        
        # 4. Создаём подписку
        from datetime import datetime, timedelta
        now = datetime.utcnow()
        await client.post(
            f"{SUPABASE_URL}/rest/v1/subscriptions",
            headers={**headers, "Prefer": "return=minimal"},
            json={
                "user_id": user_id,
                "plan": plan,
                "status": "active",
                "started_at": now.isoformat(),
                "expires_at": (now + timedelta(days=days)).isoformat()
            }
        )
    
    return {"email": email, "password": password, "user_id": user_id}
```

**На стороне приложения (уже реализовано):**

Текущий экран логина (`app/auth/login.tsx`) уже поддерживает вход по email/password. Нужно только добавить проверку подписки после входа и блокировку доступа при истёкшей подписке.

### Изменения в базе данных

Рекомендуется добавить поле `telegram_id` в таблицу `profiles` для связи Telegram-аккаунта с профилем в приложении:

### Проверка подписки в приложении

Приложение уже запрашивает подписку через `account.getSubscription`. Нужно добавить middleware, который при каждом входе проверяет `expires_at` и блокирует доступ к чатам, если подписка истекла.

## 5. Управление подписками через бота

| Действие | Что делает бот |
|----------|---------------|
| **Новая подписка** | Создаёт пользователя + профиль + подписку, отправляет логин/пароль |
| **Продление** | Обновляет `expires_at` в таблице `subscriptions` |
| **Отмена** | Устанавливает `status = 'cancelled'` в `subscriptions` |
| **Сброс пароля** | Вызывает `supabase.auth.admin.updateUserById(id, { password })` |
| **Блокировка** | Устанавливает `status = 'blocked'` или удаляет пользователя |
